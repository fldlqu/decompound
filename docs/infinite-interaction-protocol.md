# Infinite-mode interaction protocol

Infinite mode is controlled by complete input lines and has a deliberately small
transport-independent protocol.

## State machine

After initialization, the process is in `WAITING` with one valid one-component
compound displayed.

```text
WAITING -- Enter/blank line --> EXTEND --> VALIDATE --> DISPLAY --> WAITING
WAITING -- q/quit/exit      --> STOP
WAITING -- EOF              --> STOP
WAITING -- Ctrl-C           --> STOP
WAITING -- other text       --> DIAGNOSTIC --> WAITING
```

Only the `EXTEND` transition mutates the compound state.

## Input classification

`classify_input(line)` returns an immutable `InteractionInput` with one action:

- `extend`: the line contains only whitespace, including the normal `"\n"`
  generated by pressing Enter;
- `quit`: EOF (`""`) or case-insensitive `q`, `quit`, or `exit` surrounded by
  optional whitespace;
- `invalid`: any other text.

Invalid input emits a stderr diagnostic and leaves both semantic state and
stdout unchanged. This prevents accidental extensions caused by mistyped quit
commands.

## Initialization

The generator selects a rightmost head, optionally constrained by `--type`, and
immediately displays it as the first state. No Enter is required for this base
state. Its semantic component count is one.

## Extension transaction

For each Enter action:

1. select one licensed extension step for the current semantic type;
2. derive a new immutable `Compound` from the old state;
3. independently validate the new state;
4. linearize the complete new word;
5. publish it to stdout.

Validation and linearization happen before display. If either fails, the CLI
error boundary reports the failure rather than publishing a partial state.

## Stream modes

When stdout is a TTY, the first word is written without a newline and later
versions redraw that line. A normal stop writes the terminating newline.

When stdout is not a TTY, initialization and every successful Enter action emit
one complete word followed by a newline. No ANSI control sequences are emitted.
This is suitable for scripted input such as:

```bash
printf '\n\nq\n' | decompound --infinite --seed demo
```

which produces exactly three complete states: the base word and two extensions.

## Termination

EOF, quit commands, and keyboard interruption are successful user termination
and return status zero. They do not produce warning or error diagnostics.
Physical memory, terminal capacity, and process lifetime are the only general
limits; the protocol imposes no hidden extension maximum.
